Aeronautics Workshop Campaign

Design a modular aircraft, choose a workshop contract, and prove the machine on a living test range. Contracts now validate real gate crossings, stable hover, landing quality, payload survival, fuel, structural integrity and the first failure in the chain. Free-flight sandbox remains available whenever you only want to experiment.

  • Place: the workshop starts empty. Left-click the floor for the first module, then build from adjacent faces. Install exactly one oriented Command Core anywhere before launch. Its forward/up orientation defines the craft control frame.
  • Remove: right-click a block.
  • Orient module: choose its forward/chord axis, then use R and Shift+R to roll the profile. Wings now have a real aerodynamic normal.
  • Camera: hold Alt and drag, or drag with the middle mouse button, to orbit. Use the mouse wheel to zoom.
  • Flight controls: W/S requests forward or reverse thrust, Z/C requests lateral thrust, Space/Left Ctrl requests vertical thrust, ↑/↓ controls pitch, A/D or ←/→ controls yaw, Q/E controls roll, −/+ changes passive vertical thruster power, comma/period changes balloon power, both pairs are rebindable and both sliders show the current hover threshold, direct pilot inputs retain full authority, and G toggles gyro stabilization. Balloon lift weakens with altitude and now has mild vertical damping, so a fixed setting approaches its equilibrium height without excessive bouncing.
  • Contracts: complete certification, gate, courier and heavy-lift trials to earn credits and stars.
  • Control surfaces: select the cyan module, assign pitch/yaw/roll and choose automatic or manual sign before placement.
  • Gimballed engine: the purple vector thruster bends its nozzle and generates control torque without hidden forces.
  • Damage: impacts damage local modules or cargo, propagate through connections, cause fuel leaks and can tear entire disconnected branches from the craft.
  • Cargo contracts: the payload must remain attached and meet its required integrity at delivery; barely surviving cargo no longer counts as a clean success.
  • Current solver limit: blueprints may contain up to 2500 modules, but test flight is capped at 480 attached modules until collider merging is implemented.
  • Mission HUD: follow the objective, timer, integrity, lost parts and live fuel-leak rate.
  • Engineering analysis: the workshop and runtime use the same part data, power settings and control mixer.
  • Undo/redo: use the buttons or Ctrl+Z / Ctrl+Y. Blueprint files can also be exported and imported as JSON.
  • Save: craft blueprints are stored locally in the browser and migrated from older stages.
